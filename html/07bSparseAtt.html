Atencin dispersa

El proceso de enmascaramiento en la atencin permite ignorar las relaciones de las tkens actuales con los tkens futuros. Podemos ver que est el enmascaramiento est ligado a la concepcin de la atencin como un mecanismo grfico. En la atencin enmascarada podemos suponer que tenemos una grfica tal que un elemento $x_i$ est conectado con un elemento $x_j$ si y slo si $j \leq i$, siendo $i$ y $j$ la posicin absoluta de los tkens en la oracin de entrada.

Child et al. (2019) proponen extender esta concepcin todava ms para limitar las conexiones de los tkens actuales (y por tanto la informacin que pueden observar) a elementos que se encuentran en una posicin determinada con respecto al elemento actual. Los autores proponer varias formas de conectar estos elementos con respecto a las relaciones de vecindad en la grfica, de tal forma que se pueden determinar de manera especfica las vecindades $\mathcal{N}_i$ de los elementos $x_i$. De tal forma, que la matriz de pesos de atencin slo tendr valores en las entradas correspondientes a los elementos que estn conectados entre s y tendr 0 en las otras entradas, funcionando como una matriz de adyacencia de una grfica pesada. De la presencia de esto 0's es por lo que los autores llaman atencin dispersa a este mecanismo.

En general, podemos pensar a la atencin dispersa como un mtodo de auto-atencin en grficas en donde se toma en cuenta slo los elementos relacionados en una grfica no necesariamente conectada por completo. Esto es:

$$h_i = \sum_{j \in\mathcal{N}_i} \alpha_(x_i, x_j) \psi_v(x_j)$$

Para simplificar el clculo de la atencin dispersa, empero, Child et al. (2019) proponen el uso de conexiones simples, que no requieran de la especificacin explcita de una estructura de grfica. Por ejemplo, los autores proponen algunos casos particulares que se muestran a continuacin:

El caso ms sencillo es el tomar en cuenta slo los $k$ elementos anteriores al tken actual. Es decir, en lugar de relacionar todos los anteriores, se tomar una ventana de tamao $k$ para tomar los elementos. De esta forma, la representacin de un elemento $x_i$ depender de la atencin puesta slo a los $k$ elementos anteriores, a esto le llaman stride attention. En la figura anterior (c) tambin se puede ver otra versin de atencin fija. En este sentido, podemos definir la atencin dispersa como sigue:

Atencin dispersa: Es un mecanismo de auto-atencin que genera matrices de atencin dispersas, al considerar que las relaciones entre los datos de entrada $x_1, x_2,..., x_n$ no definen una grfica completamente conectada. En particular, podemos definir la atencin (Stride) dispersa, cuando las vecindades de los elementos estn dadas como: $$\mathcal{N}_i = \{j : max(0, i-k) \leq j \leq i \}$$ Donde $k$ es un hiperparmetro que determina el nmero de elementos previos que se consideran.

Para la implementacin de esta atencin dispersa utilizaremos una mscara, pero tomando en cuenta el considerar slo los $k$ elementos anteriores. El nmero $k$ de elementos previos que debemos considerar queda determinado como un hiperparmetro que llamamos stride. La implementacin es similar a la auto-atencin enmascarada, aunque tiende a ser ms costosa la creacin de la mscara.

import torch
import torch.nn as nn
import matplotlib.pyplot as plt
from seaborn import heatmap as hm
import numpy as np

class SparseAttention(nn.Module):
    #Atencin enmascarando subsecuentes
    def __init__(self, d_model, stride=3):
        super(SparseAttention, self).__init__()
        self.d_model = d_model
        self.stride = stride
        self.Q = nn.Linear(d_model, d_model, bias=False)
        self.K = nn.Linear(d_model, d_model, bias=False)
        self.V = nn.Linear(d_model, d_model, bias=False)

    def forward(self, x):
        query, key, value = self.Q(x), self.K(x), self.V(x)
        scores = torch.matmul(query, key.T)/np.sqrt(self.d_model)
        #Enmascaramiento de los scores
        mask  = self.masking(x)
        scores = scores.masked_fill(mask == 0, -1e9)
        att = nn.functional.softmax(scores, dim=-1)
        h = torch.matmul(att, value)

        return h, att

    def masking(self, x):
        #Creacin de la mscara
        n = x.size(0)
        mask = np.ones((n,n))
        for i in range(0,n):
            for j in range(0,self.stride):
                m = max(0,i-j)
                mask[i,m] = 0
        
        return torch.from_numpy(mask) == 0

Como podemos observar, el modelo de atencin dispersa considerar slo los $k$ elementos previos, ignorando los elementos subsiguientes y aquellos elementos previos que estn ms all de distancia $k$ al tken actual. Si visualizamos la matriz de atencin, podemos ver que la matriz ser dispersa, conteniendo slo valores de probabilidades en la diagonal y las $k$ entradas previas a sta. Por ejemplo, si consideramos un stride igual a 3, estaremos asumiendo una estructura grfica donde cada tken conencta slo con los 2 anteriores (y el mismo):

La matriz de atencin sera como se muestra a continuacin (donde todava no se ha aprendido los pesos adecuados).

La atencin dispersa, como lo proponen los Child et al. (2019), busca factorizar la atencin para enfocarse en los llamados modelos autoregresivos que generen secuencias tanto de texto, como de imgenes o audio. La atencin dispersa tambin busca disminuir la complejidad espacial de la atencin comn, pues es claro que una matriz de atencin requiere de una complejidad $O(n^2)$ en memoria, mientras que con la atencin dispersa, el uso de memoria (usando representaciones dispersas de las matrices de atencin) se redice a orden $O(kn)$; Los autres proponen que $k \approx \sqrt{n}$. Sin embargo, la creacin de la mscara puede tomar ms tiempo, aumentando la complejidad temporal. Por ejemplo, en este caso la mscara se crea en tiempo $O(kn)$.

Aplicacin de atencin dispersa

Para aplicar la atencin dispersa podemos utilizar el mismo ejemplo que hemos venido trabajando sobre un modelo del lenguaje. Utilizamos multi-cabezas, tomando en cada cabeza un tamao de stride incremental, de tal forma que la primera cabeza slo considera al elemento para la representacin, la segunda cabeza considera al elemento anterior, la tercera a los dos elementos anteriores, etc. Este es el nico cambo que introducimos en la implementacin:

import copy

class MultiHeadMaskAttention(nn.Module):
    def __init__(self, in_size, d_model, hidden=128, heads=3, dropout=0.3):
        super(MultiHeadMaskAttention, self).__init__()
        self.d_model = d_model
        self.enc = Encoding(in_size, d_model)
        #Uso de atencin dispersa con stride incremental
        self.att = nn.ModuleList([copy.deepcopy(SparseAttention(d_model, stride=i+1)) for i, _ in enumerate(range(heads))])
        self.lin = nn.Linear(heads*d_model, d_model, bias=True)
        self.norm = LayerNorm(d_model)
        self.ffw = nn.Sequential(nn.Linear(d_model, hidden), nn.ReLU(),
                                nn.Linear(hidden, d_model))
        self.drop1 = nn.Dropout(p=dropout)
        self.drop2 = nn.Dropout(p=dropout)
        self.drop3 = nn.Dropout(p=dropout)
    
    def forward(self, x):
        x_e = self.enc(x)
        x_e = self.drop1(x_e)
        head_att = [head(x_e) for head in self.att]
        self.att_weights = [head[1] for head in head_att]
        heads = [head[0] for head in head_att]
        multi_heads = torch.cat(heads, dim=-1)
        h = self.lin(multi_heads)
        h_norm = x_e + self.norm(h)
        h_norm = self.drop2(h_norm)
        out = self.ffw(h)
        
        return self.drop3(h_norm + self.norm(out))

Los datos de entrenamiento tienen las mismas caractersticas que anteriormente: se busca predecir la siguiente palabra, pero ahora el contexto de prediccin ser limitado por el tamao del estride; sin embargo, debe notarse que esto no estima una probabilidad de ngramas, pues la informacin se transmite entre los diferentes elementos. Por ejemplo, si el stride es 3, entonces la representacin del cuarto tken $w_4$ depender de $w_3$ y $w_2$, pero no de $w_1$. Pero tanto la representacin de $w_3$ y $w_2$ s dependen de $w_1$ por lo que la informacin de este tken se pasa de manera indirecta.

Finalmente, nuestro inters radica en explorar el tipo de matrices de atencin que se estn obteniendo. Como lo deciamos, hemos definido un stride incremental, por lo que la primera cabeza tendr una matriz diagonal. En este caso, realmente no se est haciendo un proceso de atencin, pues el elemento se est representando slo por s mismo o, de forma ms especfica, por su proyeccin en el espacio de valores; es decir, tenemos que: $h_i = \psi_v(x_i)$ con un peso de atencin de probabilidad 1. Los otros casos son ms interesante, pues en la segunda cabeza, cada elemento se representar por s mismo y el elemento anterior. En este caso particular, parece que todos las probabilidades de los pesos de atencin son cercanas a $\frac{1}{2}$; es decir, cada elemento se representa con la informacin tanto de s mismo como del elemento anterior por igual. En la cabeza 3 se representar por los 2 elementos anteriores, y finalmente en la cabeza 4 se representar por los 4 elementos anteriores, que en el ejemplo particular que aqu usamos corresponde a todos los elementos previos.
