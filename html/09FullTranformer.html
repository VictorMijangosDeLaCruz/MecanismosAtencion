Transformador

La arquitectura original de un transformador es una arquitectura encoder-decoder, cuenta con dos grandes secciones en donde la primera, el encoder, toma la cadena de entrada y la codifica a travs de un mecanismo de auto-atencin para pasar los datos codificados a un decoder, la segunda seccin del transformador, que se encargar de decodificar estos datos en una nueva cadena. Como puede imaginarse, los transformadores son de gran utilidad para la traduccin automtica y otras tareas que pasan de secuencia a secuencia.

Los transformadores han tenido un gran impacto en el procesamiento del lenguaje natural, pues permiten obtener modelos de gran capacidad que, adems, son capaces de generar lenguaje de manera efectiva. Para poder definir la arquitectura del transformador definimos el encoder y el decoder; para este ltimo requerimos de un mecanismo de atencin que tome tanto la cadena de entrada como la cadena de salida.

Atencin encoder-decoder

Al igual que los modelos de atencin en las redes recurrentes encoder-decoder, los transformadores utilizan una atencin que representa los elementos de la salida a partir de los datos de entrada, para poner atencin a aquellos elementos de entrada que tengan ms influencia en la salida actual. En este caso, se toma una query que es elemento de la salida. Supongamos que tenemos una cadena de entrada $x_1, x_2, \cdots x_n$ que se mapea en una cadena de salida $y_1, y_2, \cdots y_m$, entonces buscaremos obtener una representacin de los elementos de salida $y_1, y_2, \cdots y_m$ en trminos de la entrada $x_1, x_2, \cdots x_n$. Esta representacin pondr "atencin" a los elementos de entrada:

$$h^y_i = \sum_{j=1}^n \alpha(y_i, x_j) \psi_v(x_j)$$

Se obtendrn las representaciones para los $m$ elementos de salida. Aqu cabe decr que $\psi_v(x_j) = W_v x_j$ es una proyeccin en el espacio de valores de los datos de entrada y los pesos de atencin se estiman de manera similar, peroconsiderando que los elementos $x_j$ corresponden a las entradas y $y_i$ a las salidas. De tal forma que estos pesos de atencin se estiman como:

$$\alpha(y_i, x_j) = Softmax_x\Big( \frac{\psi_q(y_i)^T \psi_k(x_j)}{\sqrt{d}} \Big)$$

La funcin $\psi_q(y_i) = W_q y_i$ es una proyeccin al espacio de queries de los valores de las salidas, mientras que $\psi_k(x_j) = W_k x_j$ es la proyeccin al espacio de keys de las codificaciones de entrada. De tal forma que la matriz de pesos de atencin muestra las relaciones entre la entrada y la salida:

La atencin entre el encoder y el decoder es similar a la auto-atencin que se usa en cada mdulo, pero toma como valores de entrada $x_1, x_2, \cdots, x_n$ a las salidas del decoder, que ya han sido representados por medio de la auto-atencin. Asimismo, toma los tkens de salida que tmabin representa por medio de auto-atencin (enmascarada). En la implementacin de esta atencin llamamos encode a los elementos codificados del encoder.

import torch
import torch.nn as nn
import numpy as np
import copy
from tqdm import tqdm
from transformers import *

class Attention(nn.Module):
    def __init__(self, d_model):
        super(Attention, self).__init__()
        # Capas de proyecciones
        self.d_model = d_model
        self.Q = nn.Linear(d_model, d_model, bias=False)
        self.K  = nn.Linear(d_model, d_model, bias=False)
        self.V  = nn.Linear(d_model, d_model, bias=False)
        
    def forward(self, x, encode):
        # Proyeccin de los datos
        query,key,value = self.Q(x),self.K(encode),self.V(encode)
        scores = torch.matmul(query, key.T)/np.sqrt(self.d_model)
        p_attn = torch.nn.functional.softmax(scores, dim = -1)
        Vs = torch.matmul(p_attn, value).reshape(x.shape)
        
        return Vs, p_attn

Encoder

El encoder codifica la cadena de entrada en un conjunto de vectores con que se alimentar el decoder para obtener las salidas. Para hacer esta codificacin, la arquitectura que se toma cuenta con: 1) embeddings y codificacin posicional; 2) auto-atencin multi-cabeza; y 3) capa feedforward. Adems aplica sum ay normalizacin en cada capa. La siguiente figura expresa esta estructura:

En el diagrama las conexiones que saltan capas representan la suma y normalizacin. Definimos la implementacin del encoder de manera similar que lo hemos hecho en la definicin de las cabezas de auto-atencin.

class Encoder(nn.Module):
    def __init__(self, in_size, d_model, hidden=128, heads=3, dropout=0.3):
        super(Encoder, self).__init__()
        self.d_model = d_model
        self.enc = Encoding(in_size, d_model)
        self.att = nn.ModuleList([copy.deepcopy(SelfAttention(d_model)) for _ in range(heads)])
        self.lin = nn.Linear(heads*d_model, d_model, bias=True)
        self.norm = LayerNorm(d_model)
        self.ffw = nn.Sequential(nn.Linear(d_model, hidden), nn.ReLU(),
                                nn.Linear(hidden, d_model))
        self.drop1 = nn.Dropout(p=dropout)
        self.drop2 = nn.Dropout(p=dropout)
        self.drop3 = nn.Dropout(p=dropout)
    
    def forward(self, x):
        x_e = self.enc(x)
        x_e = self.drop1(x_e)
        head_att = [head(x_e) for head in self.att]
        self.att_weights = [head[1] for head in head_att]
        heads = [head[0] for head in head_att]
        multi_heads = torch.cat(heads, dim=-1)
        h = self.lin(multi_heads)
        h_norm = x_e + self.norm(h)
        h_norm = self.drop2(h_norm)
        out = self.ffw(h)
        
        return self.drop3(h_norm + self.norm(out))

Decoder

El decoder es particular; suarquitectura es ms compleja pero tambin guarda similitudes con la del encoder, pues la cadena de salida pasa por las siguientes capas: 1) embedding y codificacin posicional; 2) auto-atencin enmascarada; y 3) capa feedforward, adems de la suma y normalizacin. En primer lugar, la auto-atencin que se utiliza es enmascarada, pues se puede ver que en la etapa de inferencia, cuando predecimos cadenas que no han sido vistas previamente, se busca generar una nueva cadena a partir de predecir la palabra siguiente. Por tanto, aqu es necesario enmascarar los tkens subsiguientes en la cadena.

Durante el entrenamiento, contamos con los pares $(\mathbf{x}, \mathbf{y})$ donde conocemos la cadena de salida. Por tanto, podemos aplicar las primeras tres capas de forma similar a como se ha mostrado en la seccin de auto-atencin enmascarada. En este caso, no habr problema en la prediccin. Sin embargo en la inferencia requeriremos otra estrategia para generar los tkens de salida.

El decoder, adems de estas capas, incorporar una capa de atencin (generalmente tambin multi-cabeza) como la que hemos descrito arriba, donde tomar las codificaciones de la entrada y las utilizar para crear las representaciones de la salida a partir del mecanismo de atencin. class Decoder(nn.Module):
    def __init__(self, in_size, d_model, hidden=128, heads=3, dropout=0.3):
        super(Decoder, self).__init__()
        self.d_model = d_model
        self.enc = Encoding(in_size, d_model)
        self.self_att = nn.ModuleList([copy.deepcopy(MaskAttention(d_model)) for _ in range(heads)])
        self.att = Attention(d_model)
        self.lin = nn.Linear(heads*d_model, d_model, bias=True)
        self.norm = LayerNorm(d_model)
        self.ffw1 = nn.Sequential(nn.Linear(d_model, hidden), nn.ReLU(),
                                nn.Linear(hidden, d_model))
        self.ffw2 = nn.Sequential(nn.Linear(d_model, hidden), nn.ReLU(),
                                nn.Linear(hidden, d_model))
        self.drop1 = nn.Dropout(p=dropout)
        self.drop2 = nn.Dropout(p=dropout)
        self.drop3 = nn.Dropout(p=dropout)
    
    def forward(self, x, encode):
        x_e = self.enc(x)
        x_e = self.drop1(x_e)
        head_att = [head(x_e) for head in self.self_att]
        self.att_weights = [head[1] for head in head_att]
        heads = [head[0] for head in head_att]
        multi_heads = torch.cat(heads, dim=-1)
        h = self.lin(multi_heads)
        h_norm = x_e + self.norm(h)
        h_norm = self.ffw1(h_norm)
        h_norm = h + self.norm(h_norm)
        h_norm = self.drop2(h_norm)
        enc_dec, self.enc_dec_att = self.att(h_norm, encode)
        enc_dec = h_norm + self.norm(enc_dec)
        out = self.ffw2(enc_dec)
        
        return self.drop3(h_norm + self.norm(out))

Encoder-decoder

Ya que hemos definido de manera separada el encoder y el decoder, la arquitectura final incorporar ambos conectando la salida del encoder con la atencin en el decoder que, adems, toma la salida de la representacin como query. El resultado de esta ltima aplicacin de la atencin entre los elementos del encoder y del decoder pasar por una capa feedforward (adems de la suma y normalizacin) para obtener las representaciones finales. Finalmente, para obtener las probabilidades de salida para las predicciones aplicamos una capa lineal y una activacin con la funcin Softmax. De esta forma, obtendremos probabilidades de una cadena de salida con la que podremos hacer predicciones. El siguiente diagrama expresa la arquitectura del transformer.

Cabe sealar que originalmente Vaswani et al. (2017) proponen que se creen copias tanto del encoder, como del decoder. En este cado, no aplicamos estas copias, slo utilizamos un encoder y un decoder. Asimismo, no hemos declarado multi-cabezas en la atencin entre el encoder y el decoder, esto con motivos de simplificar la implementacin. En esta implementacin ademas definimos funciones de encode y decode que utilizaremos para la inferencia.

In[4]:
class EncoderDecoder(nn.Module):
    def __init__(self, in_size, out_size, d_model, hidden=128, heads=5, dropout=0.3):
        super(EncoderDecoder, self).__init__()
        self.d_model = d_model
        self.encoder = Encoder(in_size, d_model, hidden=hidden, heads=heads, dropout=dropout)
        self.decoder = Decoder(out_size, d_model, hidden=hidden, heads=heads, dropout=dropout)
        self.generator = nn.Sequential(nn.Linear(d_model, out_size), nn.Softmax(1))
        
    def forward(self, x, y):
        enc = self.encode(x)
        out = self.decode(y, enc)
        
        return out

    def encode(self, x):
        return self.encoder(x)
    
    def decode(self, x, encode):
        return self.generator(self.decoder(x, encode))

Inferencia

Para realizar la prediccin de nuevas cadenas a partir de datos novedosos de traduccin, definimos un decodificador ambicioso, que es un simple decodificador que en cada tiempo tomar el tken con la probabilidad ms alta como el tken siguiente. Este tken se incorpora a la cadena predica para volver a estimar otra palabra aplicando el decoder. Como se puede ver en el cdigo, el encoder se aplica una sola vez (utilizamos la funcin encode), mientras que el decoder se aplica tantas veces como la longitud de la cadena de salida. En cada paso se incopora la palabra predicha anteriormente. Al inicio slo tomamos el smbolo de inicio de palabra. De esta forma, se puede predecir una cadena de longitud arbtrario para la salida del transformador. Asimismo, definimos funciones auxiliares que nos permitirn pasar de las palabras en los lenguajes de entrada y salida a ndices y viceversa.

def greedy_decode(model, x, max_len, start_symbol):
    encode = model.encode(x)
    ys = torch.ones(1).fill_(start_symbol).type_as(x.data)
    for i in range(max_len-1):
        prob = model.decode(ys, encode)
        _, next_word = torch.max(prob, dim = 1)
        next_word = next_word[-1].reshape(1)
        ys = torch.cat((ys, next_word), dim=0)
        
    return ys

tgt_voc_rev = {k:v for v,k in tgt_voc.items()}
def translate(sent, model, max_len=10):
    x_sents = []
    for w in sent.split():
        idx_w = src_voc[w]
        x_sents.append(idx_w)
    x_sents = [bos] + x_sents + [eos]
    y = greedy_decode(model, torch.tensor(x_sents).to(device), max_len, bos)
    sent = y.cpu().detach().tolist()
        
    return ' '.join([tgt_voc_rev[word] for word in sent][1:])
