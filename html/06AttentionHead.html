Cabeza de auto-atencin

En los transformadores se suelen utilizar diferentes cabezas de atencin para procesar los datos de entrada. Una cabeza de atencin es simplemente la aplicacin de una capa de atencin aplicada a los datos de entrada ya codificados por medio de los embeddings y la codificacin posicional. Dentro de la red que construiremos se contar entonces con los siguientes elementos:

  1. Embeddings y codificacin posicional: Representarn los datos en formato vectorial para que pueden entrar dentro de las cabezas de auto-atencin.
  2. Cabeza de auto-atencin: Representa los datos a partir de la auto-atencin como $h_i = \sum_j \alpha(x_i, x_j) \psi_v(x_j)$.
  3. Suma y normalizacin: Normaliza los datos con normalizacin de lote y suma capas previas de manera residual.
  4. Feedforward: Finalmente, aplica una capa de red feedforward con activacin ReLU. Esta capa puede verse como $$x_i = W ReLU(W'h_i + b') + b$$ donde $W$ y $W'$ son pesos, $b$ y $b'$ los bias y $ReLU(x) = \max\{0, x\}$.

El resultado de todo este proceso genera nuevos vectores que representan a la entrada. Dentro del transformador, el proceso que estamos describiendo corresponde al codificador con una sola cabeza de atencin. Una sola cabeza de atencin realiza la operacin de auto-atencin como se ha revisado previamente. Ilustramos la forma en que construimos el mdulo:

Considereamos slo una cabeza de atencin que se conforma de la proyeccin hacia el espacio de queries, keys y values; las dos primeras se multiplican para obtener los pesos de atencin con ayuda de softmax, y con esto se hace la suma convexa de los valores para obtener las nuevas representaciones.

import torch
import torch.nn as nn
from transformers import *

class AttentionModule(nn.Module):
    def __init__(self, in_size, d_model, hidden=128, dropout=0.3):
        super(AttentionModule, self).__init__()
        self.d_model = d_model
        #Embedding y codificacin posicional
        self.enc = Encoding(in_size, d_model)
        #Auto-atencin
        self.att = SelfAttention(d_model)
        #Normalizacin
        self.norm = LayerNorm(d_model)
        #Red feedforward
        self.ffw = nn.Sequential(nn.Linear(d_model, hidden), nn.ReLU(),
                                nn.Linear(hidden, d_model))
        #Dropouts
        self.drop1 = nn.Dropout(p=dropout)
        self.drop2 = nn.Dropout(p=dropout)
    
    def forward(self, x):
        x_e = self.enc(x)
        h, self.p_mat = self.att(self.drop1(x_e))
        h_norm = x_e + self.norm(h)
        out = self.ffw(self.drop2(h))
        
        return h_norm + self.norm(out)

Los datos textuales son procesados para obtener vectores de ndices. Estos vectores se pasan a formato torch y se crean los datos de entrenamiento. El objetivo es predecir el tken siguiente dado el anterior. Para esto, los pares de entrenamiento consisten en las cadenas de entrada alineadas en un tken siguiente. Esto es, la prediccin que buscamos es como:

$$[bos] w_1 w_2 ... w_n \mapsto w_1 w_2 ... w_n [eos]$$

Donde los smbolos especiales [bos] y [eos] indican inicio y final de cadena, respectivamente. Por tanto, en cada tken se busca predecir el siguiente.

Atencin Multi-cabeza

La arquitectura de los transformadores propone el uso de mltiples cabezas para procesar la informacin de entrada. El concepto de multi-cabeza busca tener varias psoibles representaciones que capturen diferentes relaciones entre los elementos de entrada del modelo. Para ayudar al modelo a hacer mejores predicciones se pueden utilizar mltiples cabezas, de tal forma que cada cabeza puede aprender un conjunto de relaciones distinto y utilizar ese conjunto de relaciones para poder estimar diferentes representaciones de los datos de entrada. As cada cabeza se enfocara a una representacin particular, permitiendo una mayor variacin en stas.

Las mltiples cabezas no se calculan de forma secuencial, como suele hacerse con las capas, sino que se estiman de forma paralela. Es decir, se calculan $N$ veces la auto-atencin. Esto implica realizar cada paso de la auto-atencin $N$ veces. De tal forma que, para distinguir la cabeza, podemos utilizar un sub-ndice:

$$head_i = Att(Q_i, K_i, V_i) = Softmax\Big( \frac{Q_i K_i^T}{\sqrt{d_i}} \Big) V_i$$

Para cada $i \in \{1,2,...,N\}$. Entonces, gneramos $N$ proyecciones de query, key y value. Asimismo, obtenemos $N$ matrices de atencin. Esto nos dar un conjunto de $N$ salidas, matrices de $n \times d$ ($n$ elementos de entrada por $d$ dimensin del modelo). Para poder reducir esto a un slo conjunto de datos, combinamos la salida de cabezal a partir de la concatenacin de las salidas de todos los cabezales y multiplicando por una matriz de $N d \times d$. Esto es, la representacin final ser:

$$h = [head_1 || head_2 || \cdots || head_N] W + (b)$$

Donde $[ \cdot || \cdot]$ es la operacin de concatenacin y $W \in \mathbb{R}^{Nd \times d}$$. El siguiente diagrama muestra el cmo se realiza esta operacin; en ste se observa que se realizan $N$ proyecciones (query, key, value) de los datos de entrada, y que existen $N$ capas paralelas de atencin:

La auto-atencin multi-cabeza puede verse como una combinacin afn de las representaciones obtenidas en las cabezas de auto-atencin. Supongamos que la matriz $W$ tienen $N$ secciones $W_i \in \mathbb{R}^{d\times d}$ tal que $W = [W_1 || W_2 || \cdots || W_N]$$, entonces, la representacin de la frmula anterior puede expresarse como:

$$h = head_1 W_1 + head_2 W_2 + \cdots + head_N W_N + (b)$$

Es decir, la auto-atencin multi-cabeza est combinando la representacin de las diferetes cabezas por medio de una suma ponderada para obtener una nueve representacin que incorpore la informacin en cada cabeza. Esto en general obtiene mejores representaciones y es el estndar para los modelos de transformadores y los modelos del lenguaje.

Para nuestra implementacin multi-cabeza, haremos $N$ copias de la cala de auto-atencin (SelfAttention) utilizando nn.ModuleList de Pytorch. Esto nos permitir aplicar de forma paralela cada cabeza de auto-atencin a los datos. Guardaremos una lista de las diferentes matrices de atencin en cada cabeza para observar qu relaciones aprende en cada una.

import copy

class MultiHeadAttention(nn.Module):
    def __init__(self, in_size, d_model, hidden=128, heads=3, dropout=0.3):
        super(MultiHeadAttention, self).__init__()
        self.d_model = d_model
        #Embedding y codificacin posicional
        self.enc = Encoding(in_size, d_model)
        #Auto-atencin
        self.att = nn.ModuleList([copy.deepcopy(SelfAttention(d_model)) for _ in range(heads)])
        #Capa linear para multi cabezas
        self.lin = nn.Linear(heads*d_model, d_model, bias=True)
        #Normalizacin
        self.norm = LayerNorm(d_model)
        #Red feedforward
        self.ffw = nn.Sequential(nn.Linear(d_model, hidden), nn.ReLU(),
                                nn.Linear(hidden, d_model))
        self.drop1 = nn.Dropout(p=dropout)
        self.drop2 = nn.Dropout(p=dropout)
        self.drop3 = nn.Dropout(p=dropout)
    
    def forward(self, x):
        x_e = self.enc(x)
        x_e = self.drop1(x_e)
        head_att = [head(x_e) for head in self.att]
        self.att_weights = [head[1] for head in head_att]
        heads = [head[0] for head in head_att]
        multi_heads = torch.cat(heads, dim=-1)
        h = self.lin(multi_heads)
        h_norm = x_e + self.norm(h)
        h_norm = self.drop2(h_norm)
        out = self.ffw(h)
        
        return self.drop3(h_norm + self.norm(out))

Otras propuestas de multi-cabeza

Otras propuestas se han realizado para realizar la operacin de las multi-cabezas. En la versin propuesta por Vaswabi et ak. (2017) las cabezas de atencin trabajan de manera independiente. En contrates con esto, Cordonnier et al. (2020) proponen un conjunto de cabezas de atencin colaborativas. Estas cabezas utilizan una sola proyeccin hacia los espacios de query y key, y $N$ proyecciones hacia el espacio de values. Es decir, se tiene slo una matriz de query $W_q$ y una sola de key $W_k$. De tal forma que las matrices $Q$ y $K$ son nicas. Pero las matrices de value son $N$, $V_i$.

Para realizar diferentes matrices de atencin en las diferentes cabezas, se crea una matriz diagonal $M \in \mathbb{R}^{d\times d}$ que realiza diferentes productos puntos. De tal forma que la atencin colaorativa define las siguientes cabezas:

$$head_i = Att(Q M, K, V_i) = Softmax\Big( \frac{Q M K^T}{\sqrt{d}} \Big) V_i$$

En este sentido, el producto punto est determinado como:

$$(Q S K^T)_{i,j} = \sum_l m_l \phi_q(x_i)_l \phi_k(x_j)_l$$

Es decir, el produco entre cada vector de proyeccin est ponderado por los valores $m_1,...,m_d$ que dependen de la cabeza actual. De esta forma, en lugar de aprender $N$ matrices de query y $N$ de keys, se aprende slo un vector de los valores de la diagonal en $M_i$.
