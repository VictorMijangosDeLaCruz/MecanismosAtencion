Auto-atencin enmascarada

La auto-atencin tiene la caracterstica que, al asumir una grfica completamente conectada, toma en cuenta para la representacin de un elemento, todos los elementos que aparecen en su contexto, incluyendo aquellos elementos que aparecen posteriormente a este elemento. En la matriz de pesos de atencin podemos ver que todas las entradas tienen un peso de atencin asignado, lo que implica que existe una relacin entre los elementos, incluso los elementos futuros. Esto es, si tenemos un tken $w_i$, en la capa de auto-atencin se asumir que tiene una relacin con cada elemento $w_{i+t}$, con $t \in \{1,...,n-i\}$. Sin embargo, esto puede ser problemtico en varios casos, pues cmo hemos visto en los ejemplos anteriores, al predecir la palabra siguiente, la cabeza de atencin pondr atencin en esta palabra, en tanto tiene la capacidad de observarla.

Para evitar este sesgo, se utiliza un enmascaramiento de las palabras subsecuentes. El enmascaramiento es un procedimiento importante que ayuda a definiri modelos del lenguaje y otros procedimientos cuando no se cuenta (o no se quiere que la mquina cuente) con informacin a futuro:

Enmascaramiento: El enmascaramiento reemplaza un tken $w_i$ por una etiqueta MASK para que el modelo no tenga informacin de esta etiqueta. Este proceso oculta un tken para que no sea accesible a la auto-atencin.

El enmascaramiento tiene varias aplicaciones. Por ejemplo, en modelos de lenguaje como BERT (Devlin et al., 2019) utilizan el enmascaramiento para ocultar ciertas palabras que despes pueden ser predichas por el modelo. En este ejemplo, nos enfocaremos a enmascarar las palabras que se encuentran posteriormenta a unl tken actual. De tal forma que en la matriz de pesos de atencin slo consideraremos los pesos de la parte triangular inferior de la matriz:

El primer tken no se relacona ms que consigo mismo, el segundo slo consigo mismo y con el tken previo, y asi consecutivamente, de tal forma que cada tken se relaciona nicamente con los tkens previos. En este caso, los pesos que representan las relaciones con los tkens siguientes se interpretan como $-\infty$, esto debido a que el softmax de menos infinito es 0; de esta forma, podemos observar que $\alpha(x_i, x_{j}) = 0$ si $j > i$. De esta forma, la representacin de un elemento slo depender de los elementos previos, pues el peso de atencin 0 ignorar los elementos siguientes.

Para implementar este tipo de enmascaramiento utilizamos una matriz con valores booleanos, de tal forma que slo se conserven aquellos valores de la parte diagonal inferior. Los elementos de la parte diagonal superior los llenamos con valores grandes (lo suficientemente grandes para que su softmax sea 0).

De esta forma, definimos una clase que contenga la auto-atencin enmascarada a partir de generar una matriz que slo contenga valores en la parte triangular inferior. Por lo dems, el mecanismo de atencin es similar a la auto-atencin definida en la seccin de auto-atencin comn.

import torch
import torch.nn as nn
import numpy as np

class MaskAttention(nn.Module):
    #Atencin enmascarando subsecuentes
    def __init__(self, d_model):
        super(MaskAttention, self).__init__()
        self.d_model = d_model
        self.Q = nn.Linear(d_model, d_model, bias=False)
        self.K = nn.Linear(d_model, d_model, bias=False)
        self.V = nn.Linear(d_model, d_model, bias=False)

    def forward(self, x):
        query, key, value = self.Q(x), self.K(x), self.V(x)
        scores = torch.matmul(query, key.T)/np.sqrt(self.d_model)
        #Enmascaramiento de los scores
        mask  = self.masking(x)
        scores = scores.masked_fill(mask == 0, -1e9)
        att = nn.functional.softmax(scores, dim=-1)
        h = torch.matmul(att, value)

        return h, att

    def masking(self, x):
        #Creacin de la mscara
        n = x.size(0)
        subsequent_mask = np.triu(np.ones((n, n)), k=1).astype('uint8')
        
        return torch.from_numpy(subsequent_mask) == 0

Podemos definir un mdulo con auto-atencin enmascarada para generar la prediccin del tken siguiente en una cadena; es decir, un modelo del lenguaje auto-regresivo. El modelo del lenguaje con enmascaramiento es la forma comn de realizar los modelos del lenguaje, pues adiferencia de la implementacin anterior, en este caso, el modelo no tiene informacin de las palabras subsiguientes, lo que cumple de manera realista con la tarea de la prediccin de los tkens subsiguientes. El nico cambio que realizamos con la implementacin sin enmascaramiento es la incorporacin de la auto-atencin enmascarada.

import copy

class MultiHeadMaskAttention(nn.Module):
    def __init__(self, in_size, d_model, hidden=128, heads=3, dropout=0.3):
        super(MultiHeadMaskAttention, self).__init__()
        self.d_model = d_model
        #Embedding y codificacin posicional
        self.enc = Encoding(in_size, d_model)
        #Auto-atencin enmascarada
        self.att = nn.ModuleList([copy.deepcopy(MaskAttention(d_model)) for _ in range(heads)])
        #Capa linear para multi cabezas
        self.lin = nn.Linear(heads*d_model, d_model, bias=True)
        #Normalizacin
        self.norm = LayerNorm(d_model)
        #Red feedforward
        self.ffw = nn.Sequential(nn.Linear(d_model, hidden), nn.ReLU(),
                                nn.Linear(hidden, d_model))
        self.drop1 = nn.Dropout(p=dropout)
        self.drop2 = nn.Dropout(p=dropout)
        self.drop3 = nn.Dropout(p=dropout)
    
    def forward(self, x):
        x_e = self.enc(x)
        x_e = self.drop1(x_e)
        head_att = [head(x_e) for head in self.att]
        self.att_weights = [head[1] for head in head_att]
        heads = [head[0] for head in head_att]
        multi_heads = torch.cat(heads, dim=-1)
        h = self.lin(multi_heads)
        h_norm = x_e + self.norm(h)
        h_norm = self.drop2(h_norm)
        out = self.ffw(h)
        
        return self.drop3(h_norm + self.norm(out))

devoc = {i:t for t,i in voc.items()}
def result(text, model):
    #Funcin para predecir la siguiente palabra dado el contexto
    tokens = text.split()
    x = torch.tensor([voc[t] for t in tokens])
    pred = model(x)
    max_token = pred.argmax(axis=1).detach().numpy()
    
    return pred.detach().numpy(), ' '.join([devoc[i] for i in max_token])

p, pred_text = result('[bos]', model)
print('Palabra siguiente con mayor prob: {}'.format(pred_text))

#Visualizacin de probabilidades ms altas
args = np.argsort(p[-1])[::-1]
